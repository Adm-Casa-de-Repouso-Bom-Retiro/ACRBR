## Item 1: Descrição do Projeto
O Administrativo Casa de Repouso Bom Retiro (ACRBR) é um sistema web de gerenciamento desenvolvido para atender às necessidades operacionais e administrativas do lar de idosos (casa de repouso). O sistema tem como objetivo centralizar e organizar as informações dos residentes, profissionais de saúde, rotinas de cuidado, proporcionando mais eficiência, segurança e qualidade no atendimento prestado aos idosos.

---

## Item 2: Situação Problema (Cenário)
Muitos lares de idosos no Brasil ainda operam com processos predominantemente manuais, o que gera uma série de problemas no cotidiano dessas instituições. Informações sobre saúde, histórico médico e alergias dos residentes ficam registradas em fichas físicas, sujeitas à perda ou deterioração, dificultando o acesso em situações de emergência. O controle de medicamentos também é feito de forma manual, sem alertas automáticos, aumentando o risco de doses esquecidas ou administradas incorretamente. Soma-se a isso a dificuldade na gestão da equipe, com escalas e turnos organizados de maneira informal, o que pode gerar falhas na cobertura de cuidados. A ausência de um sistema digital impede ainda o acompanhamento do histórico de saúde e das ocorrências de cada residente ao longo do tempo, comprometendo a qualidade e a continuidade do atendimento.

---

## Item 3: Descrição da Proposta
O sistema ACRBR surgiu do desejo da equipe de desenvolver um projeto com propósito real, que fosse além de um trabalho acadêmico e pudesse gerar impacto positivo em uma comunidade próxima. A conexão com a causa veio de forma natural: a mãe de uma das integrantes do grupo fornece produtos alimentícios e de higiene básica para um lar de idosos da região, e por meio dessa relação comercial com o proprietário, foi possível conhecer de perto as dificuldades enfrentadas pela instituição no dia a dia. Em conversas informais, o dono do lar relatou os desafios de gerenciar os cuidados dos residentes sem nenhum suporte tecnológico, dependendo exclusivamente de anotações manuais e da memória da equipe para controlar medicamentos, escalas e informações de saúde. Essa realidade motivou o grupo a propor uma solução concreta e direcionada às necessidades reais dessa instituição. Assim, o ACRBR foi pensado não como um sistema genérico, mas como uma ferramenta desenvolvida a partir de um problema real, identificado por pessoas reais, para ajudar quem cuida de pessoas que merecem atenção e dignidade.

## Requisitos Funcionais
RF001 - O sistema deve permitir manter o cadastro de residentes(Dados: Nome, Idade, CPF, telefone para contato)  
RF002 - O sistema deve permitir manter o cadastro de profissionais de saúde (Dados: Nome, CPF, telefone, formação)  
RF003 - O sistema deve garantir que todos os residentes possuam uma ficha médica.  
RF004 - O sistema deve permitir aos profissionais de saúde, acesso às fichas médicas dos residentes.  
RF005 - O sistema deve manter informações(Dados: Residentes, profissionais de saúde.  
RF006 - O sistema deve permitir organizar informações (Dados: Residentes, profissionais de saúde.  
RF007 - O sistema deve permitir cadastrar e atualizar informações na ficha médica dos residentes (Dados: histórico, alergias, observações).  
RF008 - O sistema deve permitir o registro do uso de medicamentos dos residentes do lar.  
RF009 - O sistema deve permitir registrar observações relacionadas aos residentes.  
RF010 - O sistema deve permitir consultar o histórico de cada residente.  

---

## Requisitos Não Funcionais
RNF001 - O sistema deve ser acessível via navegador web (ex: Chrome, Edge, FireFox).  
RNF002 - O sistema deve possuir cores agradáveis na interface.  
RNF003 - O sistema deve ser de fácil uso.  
RNF004 - O sistema deve permitir salvar as informações sem perder dados já cadastrados.
